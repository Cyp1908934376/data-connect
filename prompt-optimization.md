# 数据对接服务开发提示词（完整技术规范）

## 技术环境
- 后端：JDK 8+（最低要求JDK 8）
- 前端：现代浏览器支持（Chrome 80+、Firefox 75+、Edge 80+）
- 架构：前后端不分离，服务端渲染（FreeMarker模板引擎），单体应用
- 部署方式：单个JAR包直接运行（内嵌Tomcat），无需额外配置，`java -jar app.jar` 即可启动
- 元数据存储：H2嵌入式数据库（文件模式，零环境依赖，数据文件自动生成在应用目录下）
- 数据源配置：全部动态化，通过Web界面管理，存储在H2中，无需手动编辑任何配置文件

## UI布局结构
**整体布局：** 经典企业管理后台设计
- 左侧菜单栏：数据源管理、模板管理、对接流程、任务管理，支持折叠/展开
- 顶部导航栏：系统Logo+名称、面包屑导航
- 内容区域：根据菜单切换显示对应模块内容，支持标签页多窗口模式

## 数据处理逻辑设计

### 接口数据处理架构
**三层处理模式：**

**第一层：输入接口处理**
```
职责：处理接口原始返回数据，转换为标准格式

输入：
- 原始响应数据（response）
- HTTP状态码（statusCode）
- 响应头（headers）

可用变量：
- response: 原始响应数据
- statusCode: HTTP状态码
- headers: 响应头对象
- config: 接口配置信息

处理场景：
1. 标准JSON响应：直接解析为JSON对象
2. XML响应：转换为JSON对象
3. HTML表格：解析表格数据为JSON数组
4. CSV文本：解析为JSON数组
5. 二进制数据：Base64编码或自定义解析
6. 分页数据：自动处理分页逻辑，合并多页数据
7. 需要登录的接口：处理登录逻辑和会话保持
8. 需要签名的接口：处理签名生成

输出要求：
- 必须返回JSON对象或JSON数组
- 统一数据结构：{ data: [...], meta: {...} }
- meta包含：总数、页码、其他元数据
```

**第二层：核心处理逻辑**
```
职责：数据转换、验证、清洗

处理流程：
1. 数据验证
   - 必填字段检查
   - 字段类型检查
   - 字段格式验证（邮箱、手机号、身份证等）
   - 业务规则验证

2. 数据清洗
   - 空值处理（删除、填充默认值）
   - 重复数据处理
   - 数据格式统一（日期格式、数字精度）
   - 特殊字符处理（HTML转义、SQL注入防护）

3. 数据转换
   - 字段映射（输入字段 → 输出字段）
   - 数据类型转换
   - 值域映射（如：1→启用，0→禁用）
   - 数据聚合/拆分
   - 数据计算（字段间运算、统计计算）

4. 模板驱动
   - 应用数据对接模板
   - 执行模板中的转换逻辑
   - 支持自定义函数扩展

错误处理：
- 单条数据错误不影响其他数据处理
- 错误数据记录到错误日志
- 支持错误数据重试或跳过
```

**第三层：输出接口处理**
```
职责：格式化输出数据，适配目标系统

输入：
- 处理后的数据（JSON格式）
- 输出数据源配置

处理场景：
1. 数据库输出：
   - 生成INSERT/UPDATE/DELETE SQL语句
   - 支持批量插入（提高性能）
   - 支持事务控制（全部成功或全部回滚）
   - 字段类型适配（JSON→数据库类型）
   - 主键冲突处理（跳过、更新、报错）

2. 接口输出：
   - 构造HTTP请求体
   - 处理请求头和认证
   - 支持批量请求（如果目标接口支持）
   - 请求失败重试机制
   - 限流控制（避免触发目标接口限流）

3. 文件输出：
   - 支持多种文件格式（CSV、Excel、JSON、XML）
   - 文件编码设置
   - 文件大小限制（超大文件分片处理）
   - 文件命名规则（时间戳、序列号等）

特殊情况处理：
- 目标字段不存在：自动创建或跳过
- 字段类型不匹配：自动转换或记录错误
- 字段长度超限：截断或报错
- 目标系统维护：检测到维护状态，自动暂停任务
```

### 多接口协作场景
```
场景描述：某些复杂场景需要调用多个接口才能完成数据对接

协作模式：
1. 串行协作：按顺序调用多个接口，后一个接口依赖前一个接口的结果
   - 示例：先调用认证接口获取Token，再调用数据接口

2. 并行协作：同时调用多个接口，汇总所有接口的结果
   - 示例：同时调用多个数据源，合并数据后处理

3. 主辅协作：主接口获取主要数据，辅助接口补充详细信息
   - 示例：主接口获取订单列表，辅助接口获取每个订单的详细信息

实现方式：
1. 流程配置界面支持添加多个接口步骤
2. 每个步骤可配置接口调用条件和结果处理逻辑
3. 支持步骤间的数据传递和依赖关系
4. 可视化流程图显示接口调用关系

配置示例：
{
  "steps": [
    {
      "name": "获取Token",
      "dataSource": "认证接口",
      "condition": "always",
      "resultMapping": {"token": "accessToken"},
      "next": "获取数据"
    },
    {
      "name": "获取数据",
      "dataSource": "数据接口",
      "condition": "token存在",
      "headers": {"Authorization": "Bearer ${token}"},
      "resultMapping": {"data": "items"},
      "next": "获取详情"
    },
    {
      "name": "获取详情",
      "dataSource": "详情接口",
      "condition": "data不为空",
      "forEach": true,  // 对每条数据调用
      "resultMapping": {"details": "itemDetails"},
      "next": "输出"
    }
  ]
}
```
